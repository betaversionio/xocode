# Project Config Reference

`xo.config.json` lives in the root of your project. It stores project-level settings that generators read and write.

---

## Example

```json
{
  "template": "next-app",
  "framework": "next",
  "packageManager": "pnpm",
  "ui": {
    "componentsDir": "src/components/ui",
    "hooksDir": "src/hooks",
    "libDir": "src/lib"
  },
  "database": {
    "provider": "postgres",
    "url": "postgresql://localhost:5432/mydb"
  },
  "auth": {
    "provider": "jwt",
    "secret": "{{env.JWT_SECRET}}"
  }
}
```

---

## Top-Level Fields

### `template`

Written automatically by `xo create`. Records which template scaffolded the project.

```json
{ "template": "next-app" }
```

Used to:
- Let `xo add` validate feature compatibility (e.g. block a `nestjs/` feature on a `flutter-app` project)
- Give generators context about the project's origin
- Display project info in CLI output

---

### `framework`

Auto-detected or manually set. Used by generators to conditionally run actions.

```json
{ "framework": "next" }
```

Possible values: `next`, `react`, `nestjs`, `flutter`, `express`, `django`, etc.

---

### `packageManager`

Auto-detected from lockfile presence. Used by generators when running install commands.

```json
{ "packageManager": "pnpm" }
```

Possible values: `npm`, `yarn`, `pnpm`, `bun`

---

## Namespace Blocks

Config is grouped by namespace. Any generator can define its own namespace.

### `ui`

```json
{
  "ui": {
    "componentsDir": "src/components/ui",
    "hooksDir": "src/hooks",
    "libDir": "src/lib"
  }
}
```

### `database`

```json
{
  "database": {
    "provider": "postgres",
    "url": "postgresql://localhost:5432/mydb"
  }
}
```

### `auth`

```json
{
  "auth": {
    "provider": "jwt"
  }
}
```

---

## Auto-Population

When a generator declares a `requires` field, xo checks if that key exists in `xo.config.json`. If it is missing, xo prompts the user and writes the answer back automatically.

```json
{
  "requires": ["ui.componentsDir"]
}
```

You never need to manually edit `xo.config.json` — it builds up as you run generators.

---

## State File

xo also maintains `.xo/state.json` separately from config. This tracks which generators have been installed and supports `xo undo`.

```json
{
  "installedFeatures": [
    "ui/button",
    "payment/stripe",
    "auth/jwt"
  ]
}
```

Do not edit this file manually.
